Quiero crear una aplicación web tipo CRM para gestionar prospección comercial B2B de La Bercianita, una tienda de productos gourmet artesanos de El Bierzo.

La aplicación debe servir para que un agente externo llamado OpenClaw pueda:
1. Buscar potenciales clientes.
2. Subir leads a la base de datos mediante API.
3. Consultar datos existentes si necesita evitar duplicados.
4. Actualizar el estado de cada lead.
5. Crear borradores de email comercial.
6. Registrar eventos de contacto, respuestas, tareas y bajas.

IMPORTANTE:
- No uses datos mock.
- No uses arrays hardcodeados para pintar leads.
- Toda la información debe salir siempre de la base de datos.
- Si la base de datos está vacía, la interfaz debe mostrar estados vacíos reales tipo “No hay leads todavía”.
- La app debe tener login obligatorio.
- Crea un usuario inicial:
  - usuario: admin
  - contraseña: admin123
- La contraseña debe guardarse hasheada en la base de datos, no en texto plano.
- Todas las páginas privadas deben requerir sesión.
- No dejes el dashboard accesible sin login.

Stack recomendado:
- Next.js o React + backend Node/Express.
- Base de datos PostgreSQL si está disponible en Replit.
- ORM Prisma si encaja bien.
- Tailwind CSS para el diseño.
- Autenticación con sesión segura o JWT httpOnly.
- API REST documentada.
- Diseño limpio, responsive y profesional.

Contexto del negocio:
La Bercianita vende productos gourmet artesanos de El Bierzo: pimientos asados, pera conferencia, castañas, mermeladas, pera al vino Mencía, crema de castañas y packs gourmet. Los clientes objetivo son tiendas gourmet, delicatessen, vinotecas, queserías, charcuterías premium, hoteles rurales, restaurantes de producto, tiendas de regalos gourmet y empresas de cestas corporativas.

3. Crear o actualizar un lead
POST /api/agent/leads/upsert

Body:
{
  "business_name": "Nombre tienda",
  "business_type": "tienda gourmet",
  "city": "Madrid",
  "province": "Madrid",
  "country": "España",
  "website": "https://...",
  "email": "info@...",
  "phone": "...",
  "instagram": "...",
  "source_url": "https://...",
  "source_notes": "Encontrado en su web oficial",
  "fit_score": 85,
  "fit_reason": "Vende productos gourmet, conservas premium y packs regalo.",
  "consent_status": "email_publico_corporativo",
  "crm_status": "nuevo",
  "priority": "alta"
}

Reglas:
- Si ya existe un lead con el mismo website o email, actualizarlo en vez de duplicarlo.
- Si existe pero los nuevos datos son mejores, completar campos vacíos.
- No sobrescribir manualmente estados avanzados como cliente_ganado, no_contactar o baja_solicitada sin lógica de seguridad.
- Devolver el lead final guardado.
